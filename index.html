El Proyecto

Necesitamos una forma sencilla de que los Sanitarios gestionen los cursos y jornadas que organizan.

Por razones de seguridad y protección de datos, no podemos usar Google Forms, ni otro software propietario.

Objetivos funcionales

Con ella se ayuda a los profesionales a crear y gestionar cursos que posteriormente se realizarán en las aulas.

También debe ayudar a los alumnos a buscar, mantenerse informados y apuntarse a cursos de manera cómoda.

Diseño

Debe cumplir unos mínimos exigibles

  1. Diferentes tipos de usuarios
  2. Ser una aplicación multidispositvo
  3. Adaptarse a las exigencias técnicas y tecnológicas del Hospital

Diseño Técnico

Tanto en el servidor como en el cliente hay que diseñar multitud de elementos.

Dado que esta es una tarea que puede provocar errores graves en la construcción de software, se utilizan los patrones.

La lógica, la estructura de los datos, la estructura del código el manejo de los datos, el envío de mensajes, el API, son sólo algunos elementos a diseñar.

Patrones en el servidor

El diseño del servidor se hizo por capas, cada una de ellas encargada de una función en particular e interactúa sólo con sus colindantes.

Esta forma de estructurar el trabajo permite reducir los problemas a problemas más pequeños y simples, permitiendo una mejor aproximación.

La API

Se utiliza para definir un patrón de implementación de sistemas en red y/o para definir sistemas distribuidos.

Es uno de los patrones más utilizados actualmente. Se fundamenta en construir aplicaciones CRUD usando algunos métodos del protocolo de red http.

DAO

Es un objeto de acceso a datos, un componente que suministra una interfaz común y uno o más o dispositivos de almacenamiento de datos.

Los objetos de acceso a datos pueden usarse para aislar una aplicación de la tecnología de persistencia subyacente

Reutilización de elementos

Permite un desarrollo de aplicaciones más ágil.

También permite la Reutilización de las diferentes partes del interfaz en varios proyectos distintos sin realizar cambios en el código.

Esto se consigue mediante los Componentes Web.

Organización y escalado

La estructura debe permitir un escalado a futuro de la aplicación.

La adición de las nuevas partes no debe interferir con las anteriores.

Esto suele realizarse mediante módulos o paquetes y estructuras en forma de árbol.

Versatilidad en el desarrollo

Debe ser sencillo migrar la aplicación a cualquier dispositivo, aunque estos no tengan navegador.

Las herramientas del entorno de desarrollo de la mayoría de frameworks permiten la construcción de aplicaciones móviles mixtas, aplicaciones de escritorio, aplicciones web progresivas, programación sobre microcontroladores, asistentes de voz, etc.,

Estructura de un Componente

Utilizando TypeScript se consiguen implementar características desables no disponibles en JavaScript. 


											Import { Component} from '@angular/core'; 
											@Component({
												selector: 'cursos',
												template: 
												`<h3>{{titulo}}</h3>
												 <ul>
												    <li *ngFor="let curso of cursos">
														{{ curso }}
													</li>
												 </ul>` })
											export class CursosComponent{
												titulo: string = “Lista de cursos”;
												cursos: string[] = [“curso1”,”curso2”,”curso3”]
											}
